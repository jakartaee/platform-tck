Jakarta EE 9 Platform TCK Requirements, Facts, and Figures

The Jakarta EE 9 Platform 8 TCK requires a minimum of 3.7 GB of free disk space:

  • Jakarta EE 8 Platform CTS (zipped): 502 MB
  • Jakarta EE 8 Platform CTS (unzipped): 785 MB
  • Space for TCK log/report files: 2 GB minimum (trace off)

The test suite bundle contains the following:

  • EARs: 2218
  • RARs: 14
  • JARs: 3689
  • WARs: 2347

  • 44,589 total tests:
    • Platform TCK : 42436
    • JAX-WS 3.0 TCK: 1703
    • SOAP 2.0 TCK : 450

The tests for XML Webservices and SOAP are available as standalone TCKs and implementers are required to run these standalone TCKs if they provide their own implementations of these APIs that override and/or replace the implementations in the Java SE platform.

Chapter 4, "Installation" in the Jakarta EE 9 Platform TCK User's Guide, explains how to install the Jakarta EE 9 Platform TCK on Linux, and Windows systems.

These are the stand alone TCKs, porting bundles, and JDK used to test the Jakarta EE 9 Compatible Implementation

Known Problems and Workarounds

Running the TCK Against Jakarta EE 9 Platform CI on Windows

When configuring and running the CTS against the Jakarta EE 9 Platform CI on Windows, please make sure to install all related software (Java SE, Jakarta EE 9 Platform CI, platform TCK, etc.) on the same drive (for example, C:). If you want to install on a drive other than C:, please be sure to change the following properties in <TS_HOME>/bin/ts.jte:

pathsep=;
ri.applicationRoot=c:
s1as.applicationRoot=c:
